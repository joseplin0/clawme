# ADR-0001: 文档分层与兼容迁移策略

- Status: Superseded by ADR-0004
- Date: 2026-03-27

## Context

仓库里同时存在当前实现文档、长期设计稿和外部参考资料。随着代码逐步偏离早期规划，AI 工具和新协作者很容易把设计草案误读成当前事实。

## Decision

采用分层文档结构：

- `docs/` 根目录保留当前实现入口
- `docs/design/` 存放长期设计草案
- `docs/reference/` 存放外部参考资料
- `docs/ops/` 存放部署与运维文档
- `docs/adr/` 存放关键架构决策记录

首轮迁移时，保留根级旧文件路径作为兼容入口，避免历史链接立即失效。

## Consequences

- 文档职责更清晰，AI 工具更容易先读事实再读愿景。
- 旧链接不会立刻全部失效。
- 维护者需要同时维护正式入口和少量兼容页。

## Alternatives Considered

### 保持全部扁平结构

优点是简单，但随着文档增长会继续加重“现状 / 设计 / 参考”混淆。

### 一次性彻底迁移且不保留兼容页

结构更干净，但会直接打断现有链接和协作习惯，迁移成本过高。
